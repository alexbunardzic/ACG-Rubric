# Contributing

Contributions to the ACG Rubric are welcome. This document describes how.

## Before opening a PR

Read, in order:

1. [`rubric/index.md`](rubric/index.md) — what the rubric is
2. [`rubric/scope.md`](rubric/scope.md) — what it is not
3. [`GOVERNANCE.md`](GOVERNANCE.md) — how it changes

## Kinds of contribution

- **Propose a new criterion** — use [`.github/ISSUE_TEMPLATE/propose-criterion.md`](.github/ISSUE_TEMPLATE/propose-criterion.md). Open the issue first, before the PR. Proposals sit for at least 21 days.
- **Challenge an existing criterion** — use [`.github/ISSUE_TEMPLATE/challenge-criterion.md`](.github/ISSUE_TEMPLATE/challenge-criterion.md).
- **Report an unassessable criterion** — use [`.github/ISSUE_TEMPLATE/report-unassessable.md`](.github/ISSUE_TEMPLATE/report-unassessable.md).
- **Fix a typo, clarify wording, improve an example** — open a PR directly, referencing the file changed.
- **Add or sanitize an evidence session** — see [`evidence/README.md`](evidence/README.md).

## Style

- Criteria and rubric files are written to be legible to a hostile reader. Prefer precise statements over reassuring ones.
- Avoid marketing language. Do not use words like "robust", "seamless", "state-of-the-art".
- Every criterion links to the evidence that justifies it.
